# BK_RakhovRI — Учебный букмекер

Симуляция букмекерской конторы на чистом PHP/MySQL без фреймворков.

---

## Стек

| Слой | Технологии |
|---|---|
| Backend | PHP 7.4+, PDO, сессии |
| База данных | MySQL 8 / MariaDB (InnoDB, utf8mb4) |
| Frontend | Vanilla JS, CSS Custom Properties (тёмная тема) |
| Сервер | Apache (XAMPP) |

---

### Ключевые принципы

- **Роли**: `user`, `bookmaker`, `analyst`, `admin`. Доступ проверяется функцией `requireRole()` в `bootstrap.php`.
- **CSRF**: токен в сессии, проверка через `hash_equals()` на каждый POST.
- **Пароли**: `password_hash(BCRYPT)`.
- **Гонки**: `SELECT ... FOR UPDATE` внутри транзакции при списании баланса.
- **Аудит**: каждое административное действие пишется в таблицу `audit_log`.

## Математика: динамические коэффициенты

Реализовано в `includes/odds.php`, вызывается после каждой принятой ставки.

### 1. Оверраунд (маржа букмекера)

При создании исходов букмекер задаёт начальные коэффициенты $k_i^{(0)}$.
Из них вычисляется **оверраунд** — константа для рынка:

$$R = \sum_{i=1}^{n} \frac{1}{k_i^{(0)}}$$

При честных котировках $R = 1$. В реальном букмекерстве $R > 1$ (например, $1.05$ — маржа 5%).
Оверраунд фиксируется при создании рынка и **не меняется** при пересчёте.

### 2. Начальные вероятности

Из начальных коэффициентов извлекаем подразумеваемые вероятности:

$$p_i^{(0)} = \frac{1}{k_i^{(0)} \cdot R}$$

По построению $\sum p_i^{(0)} = 1$.

### 3. Вес ставок

После каждой ставки считаем суммарный объём (в VCOIN) по каждому исходу:

$$V_i = \sum_{\text{ставки на исход } i} \text{сумма ставки}$$

Долю ставок на исход $i$:

$$w_i = \frac{V_i}{\sum_j V_j}$$

Если ставок ещё нет — $w_i = p_i^{(0)}$.

### 4. Blend: смешанная вероятность

Новая вероятность — линейная смесь начальной и рыночной оценок:

$$\tilde{p}_i = (1 - \alpha)\, p_i^{(0)} \;+\; \alpha\, w_i$$

Параметр $\alpha$ (`ODDS_SENSITIVITY = 0.30`) задаёт, насколько сильно ставки двигают коэффициент.  
При $\alpha = 0$ коэффициент не меняется, при $\alpha = 1$ полностью следует за ставками.

### 5. Новый коэффициент

Коэффициент восстанавливается из смешанной вероятности с сохранением оверраунда:

$$k_i^{\text{new}} = \max\!\left(1.01,\; \text{round}\!\left(\frac{1}{\tilde{p}_i \cdot R},\; 2\right)\right)$$

Нижняя граница $1.01$ защищает от технически некорректных коэффициентов.

### Пример

Рынок «Победитель матча», начальные коэффициенты: П1 = 2.00, Х = 3.20, П2 = 4.00.

$$R = \frac{1}{2.00} + \frac{1}{3.20} + \frac{1}{4.00} = 0.500 + 0.3125 + 0.250 = 1.0625$$

Поступили ставки: 600 VCOIN на П1, 300 на Х, 100 на П2.

$$w_{П1} = 0.60, \quad w_Х = 0.30, \quad w_{П2} = 0.10$$

При $\alpha = 0.30$:

$$\tilde{p}_{П1} = 0.70 \times 0.4706 + 0.30 \times 0.60 = 0.3294 + 0.18 = 0.5094$$

$$k_{П1}^{\text{new}} = \frac{1}{0.5094 \times 1.0625} \approx 1.85$$

Коэффициент П1 снизился с 2.00 до 1.85 — ставки на фаворита сделали его «дешевле», коэффициенты на остальные исходы выросли, маржа сохранилась.

### Ручной сброс

Когда букмекер вручную меняет коэффициент через форму в `event_edit.php`, оба поля (`odds` и `initial_odds`) обновляются одновременно — это устанавливает новую точку отсчёта и сбрасывает накопленное влияние ставок.

1. Скопировать папку в `htdocs/BK_RakhovRI`
2. Запустить Apache и MySQL в XAMPP
3. Открыть `http://localhost/BK_RakhovRI/setup.php` — создаст БД и тестовые данные
4. Войти: `admin@bk.test` / `password`
